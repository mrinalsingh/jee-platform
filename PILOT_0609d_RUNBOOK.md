# Pilot import — sheet 0609d (11 questions) → `problems` table

Generated by the MCQ_Building project's tagging pipeline. All 11 YAMLs are written to
`content/maths/generated/` and pass the importer's structural validation.

## 1. Set your DB connection
```bash
cd backend
echo 'DATABASE_URL=postgresql://USER:PASS@localhost:5432/jee_platform' > .env   # your Postgres
npx prisma migrate deploy            # if the schema isn't applied to this DB yet
```

## 2. Import
```bash
# from backend/
npx tsx scripts/import-yaml.ts ../content/maths/generated
```
Expect `inserted: 13` (the 11 pilot + the 2 pre-existing example problems) or a mix of
inserted/updated if you've run it before. UPSERT is keyed on `question_code`; `calibrated`
rows are never overwritten.

## 3. Verify in the DB
```sql
SELECT question_code, topic_code, subtopic_code, answer_type, authored_difficulty, status
FROM problems
WHERE question_code LIKE 'MAT.SYS.%' OR question_code LIKE 'PRB.%'
   OR question_code LIKE 'VEC.PLN.%' OR question_code LIKE 'PNC.COM.LATDIR.%'
ORDER BY question_code;
-- expect 11 new rows, all status = 'provisional'
```

## Recalibration (already built in — no action needed now)
As students attempt these in tests, rows land in `attempts` (append-only). Your nightly
empirical-rating batch fills `empirical_difficulty_by_round` / `empirical_time_by_round` and
flips `status` → `calibrated` at ≥30 attempts + human approval. Authored tags are never touched.
